## 📋 项目概览

### 项目目标
完美复刻金蝶ERP系统（烘焙行业版），实现1:1功能还原，包括界面布局、按钮、字段、业务逻辑、操作流程。

### 开发周期预估
- **总工时**：80-100小时
- **阶段数**：5个阶段
- **模块数**：25个一级菜单模块
- **子功能数**：120+个子功能

### 验收标准
1. ✅ 功能还原：界面布局、按钮、字段、业务逻辑、操作流程与金蝶系统1:1完全还原
2. ✅ 数据验证：测试数据能完整跑通业务流程，无报错、无逻辑矛盾
3. ✅ 稳定性：点击、提交、查询等操作流畅，无白屏、无崩溃、无报错
4. ✅ 同步性：GitHub仓库与Codespaces云端环境完全同步

## 📝 开发规则与流程

### 单功能开发流程（严格执行）

```
① 主动询问 → ② 金蝶截图确认 → ③ 开发备份 → ④ 功能开发 → ⑤ 错误记录 → ⑥ 同步更新 → ⑦ 测试交付
```

1. **主动询问**：开发前向我确认需求，询问不清楚的业务逻辑
2. **金蝶截图确认**：收到截图后，书面确认「已完全理解业务逻辑、界面布局、操作流程」才能开始
3. **开发备份**：每个功能开发前，自动备份当前代码版本（`git commit -m "backup: before developing [功能名]"`）
4. **功能开发**：按验收标准完成功能
5. **错误记录**：记录错误和修复方法到《问题与修复手册》
6. **同步更新**：开发完成后自动同步到GitHub（`git push origin main`）
7. **测试交付**：提供测试链接和5-10条测试数据

### 阶段性交付要求

每完成一个「模块」，主动通知验收，提供：
1. 本模块已完成的所有子功能清单
2. 完整的测试链接
3. 测试数据说明
4. 本次遇到的问题、修复方法及避免重复犯错的措施
