## 項目總覽

### 業務流程閉環
```
採購訂單 → 原料入庫 → 配方管理 → 生產計劃 → 成品入庫 → 庫存調撥 → 門店銷售 → 日結財務
```

### 模塊依賴關係
```
基礎數據層: 商品管理、供應商管理、門店管理
         ↓
採購層: 採購訂單、採購入庫
         ↓
生產層: 配方管理、生產計劃、原料領料、成品入庫
         ↓
庫存層: 庫存查詢、調撥管理、效期預警
         ↓
銷售層: POS開單、會員管理、訂單退換
         ↓
財務層: 日結報表、應收應付、成本核算
```

## 備份管理規則

### 備份命名規範
```
backup-[模塊名]-YYYYMMDD-[版本號]
```

### 備份示例
```
backup-商品管理-20260504-v1.0
backup-採購訂單-20260505-v1.0
backup-採購訂單-20260506-v1.1  # 修復版
```

### 備份存儲位置
```
erp-system/
├── backups/
│   ├── backup-商品管理-20260504-v1.0.tar.gz
│   └── ...
├── backend/
└── frontend/
```

## 開發規則速查表

### ❌ 嚴禁事項
1. 未經確認提前開發任何代碼
2. 替換整個項目或重寫基礎框架
3. 帶著錯誤繼續開發後續模塊
4. 跳過驗證直接進入下一模塊

### ✅ 必須執行
1. 每個模塊開發前創建備份
2. 基於極簡版本進行迭代開發
3. 嚴格遵守烘焙專屬規則（批次、效期、FIFO）
4. 單模塊驗證通過後再進入下一模塊

### 🔄 問題處理流程
```
發現問題 → 回滾備份 → 排查原因 → 修復問題 → 重新測試 → 確認通過
```
