## 二、佈局對比

### 優化前（SimpleLayout）
```
頂部橫向導航欄（橙色背景）
└── 主內容區（卡片式儀表板）
```

### 優化後（KingdeeLayout）
```
頂部標題欄（橙色背景）：Logo + 當前頁面 + 門店/用戶
└── 主容器
    ├── 左側側邊欄（深色 #1e1e2d）：一級菜單（圖標+文字）
    ├── 中間子菜單面板（白色）：分類展示子功能
    └── 右側主內容區（灰色 #f5f7fa）：顯示業務數據
```

## 三、核心功能特性

### 1. 左側側邊欄
- ✅ 深色背景（`#1e1e2d`）
- ✅ 圖標 + 文字菜單項
- ✅ 點擊高亮（橙色 `#ff6b35`）
- ✅ 支持收起/展開
- ✅ 垂直排列，一級菜單

### 2. 子菜單面板
- ✅ 點擊有子菜單嘅一級項後展開
- ✅ 多列網格佈局（3列）
- ✅ 功能分類展示（如「商品管理」「採購業務」）
- ✅ 每個分類下有具體功能鏈接
- ✅ 點擊子功能後跳轉並關閉面板

### 3. 菜單結構
| 一級菜單 | 子功能分類 |
|---------|-----------|
| 儀表板 | （無子菜單，直接跳轉） |
| 基礎資料 | 商品管理（商品列表/分類/單位）、供應商管理 |
| 採購管理 | 採購業務（訂單/入庫/退貨）、採購報表 |
| 生產管理 | 生產業務（計劃/領料/入庫）、配方管理 |
| 銷售管理 | 門店銷售（POS/退貨/日結）、會員門店 |
| 庫存管理 | 庫存業務（查詢/調撥/盤點）、庫存預警 |
| 財務管理 | 財務報表（銷售/利潤/成本）、應收應付 |
| 系統設置 | （無子菜單，直接跳轉） |

### 4. 響應式設計
- ✅ 桌面端：側邊欄 + 子面板 + 主內容並排
- ✅ 平板端：子面板自適應寬度
- ✅ 收起狀態：只顯示圖標，節省空間

## 四、訪問方式

### 同步代碼到 Codespaces
1. 喺 Codespace 入面，打開終端
2. 執行：`git pull origin main`
3. 如果前端已啟動，刷新瀏覽器即可

3. 等待端口轉發，點擊鏈接訪問
